# Transition to Self-Governance

**Status:** Normative transition plan

## Principle

Authority moves only when the receiving mechanism can preserve legitimacy. Premature decentralization can be capture; permanent founder control can be capture. Mininet chooses measured sovereign evolution.

## Phase 0 — Founder-guarded repository

Canonical mainline and releases are protected by the founder. Contributions may be anonymous, pseudonymous, or public. AI assists but cannot authorize.

Exit criteria:

- protected mainline and release process;
- traceability and independent review for critical work;
- documented emergency and recovery procedures;
- initial contributor and compensation process.

## Phase 1 — Maintainer-assisted bootstrap

Multiple independent maintainers hold review responsibility. Founder remains final guardian for canonical mainline.

Exit criteria:

- stable reviewer roster across major domains;
- key rotation and removal tested;
- no single engineer required for build or release;
- working anonymous/pseudonymous bounty settlement.

## Phase 2 — Working-group governance

Domain groups can accept ordinary changes. Cross-domain and constitutional changes remain centrally integrated.

Exit criteria:

- domain ownership policies encoded;
- appeal and conflict process;
- integration council with rotating membership;
- public evidence of reliable operation.

## Phase 3 — Hybrid GitHub and Mininet Forge

All proposals, reviews, builds, and merge decisions are represented as signed Forge objects. GitHub mirrors the same history and remains a familiar interface.

Exit criteria:

- development continues through a complete GitHub outage;
- Git import/export is deterministic;
- Forge approvals and provenance are enforced;
- independent operators host storage, builders, and indexers;
- anonymous/pseudonymous contribution works without GitHub identity.

## Phase 4 — Forge-primary governance

Mininet Forge is canonical. GitHub is a read-only or delayed mirror. Governance, releases, and bounties occur inside Mininet.

Exit criteria:

- community governance controls constitutional and release authority;
- founder has no unilateral canonical or release key;
- recovery does not require founder or GitHub;
- external audits and live adversarial exercises have passed.

## Phase 5 — GitHub optional or disabled

The community votes whether GitHub remains a public mirror, archival mirror, or is disabled. The decision must consider discoverability, censorship resistance, contributor privacy, and operational cost.

## No automatic transition

Calendar dates, contributor counts, token value, or popularity alone never trigger a phase change. Each transition requires evidence and a governed decision.

## Transition evidence rule

A phase changes only after its exit criteria are demonstrated under normal operation and at least one realistic failure exercise. Contributor count, elapsed time, or political demand alone does not satisfy a phase gate.
